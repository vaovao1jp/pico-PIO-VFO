# PIO_VFO2 — 128x64 ダイヤルUI版

`PIO_VFO` の発振部はそのままに、表示を **OLED 128x64** へ拡張した版。

## 表示レイアウト

```
┌────────────────────────────┐
│        ▼  (中央=現在周波数)  │  ← 上端の指示マーカ＋指示線
│   7.09  ＼│／  7.11         │  ← 円弧ダイヤル(大円の上部のみ)
│  ‾‾‾‾‾| | | |‾‾‾‾‾          │     目盛り: 1k=細 / 5k=中 / 10k=長+ラベル
│                            │     周波数を変えると目盛りが左右に回転
│       7.100.000            │  ← 周波数 (profont17, 中央)
│        STEP: 100 Hz        │  ← ステップ (5x8, 中央)
└────────────────────────────┘
```

- 円の中心は画面の下(外側)にあり、上部の弧だけが見える「昔の無線機ダイヤル」風。
- 中央の指示線が常に現在の発振周波数を指し、ダイヤル目盛りがスクロールする。
- 周波数を上げると目盛りは左へ流れる（右側に高い周波数が並ぶ）。

## 調整できるパラメータ（PIO_VFO2.ino 上部）

| 定数 | 役割 | 既定 |
|------|------|------|
| `DIAL_CY` / `DIAL_R` | 円の中心位置 / 半径。弧の頂点 y = CY−R | 90 / 86 |
| `DIAL_RAD_PER_HZ` | 周波数→回転角の感度。大きいほど目盛りが速く流れる | 5.0e-5 |
| `DIAL_TH_LIMIT` | 描画する弧の角度範囲(±rad) | 0.95 |

- 目盛りを「もっと密」にしたい → `DIAL_RAD_PER_HZ` を下げる（例 3.0e-5）。
- ラベルが詰まりすぎ/まばら → `drawTicks()` の 10kHz 判定間隔を変更。

## 配線・ビルド・校正

`PIO_VFO`（128x32版）と同じ。OLED は 128x64 の SSD1306 を I2C(GP4/GP5)へ。
ライブラリは U8g2。`CAL_PPM` も同じ手順で校正（現在 16.43 を設定済み）。

> OLED ドライバが SH1106 等の場合は U8g2 のコンストラクタを
> `U8G2_SH1106_128X64_NONAME_F_HW_I2C` などに変更すること。
